# Data Contract: Variant 14

## 1. Схема Normalized (data/normalized)
| Поле | Тип | Описание |
|---|---|---|
| `year` | int | Год (из поля date в API) |
| `value` | float | Значение выбросов |
| `country_iso3` | string | Код USA |

## 2. Владение и ответственность
- **Owner:** Студент (разработчик ETL-процесса).
- **Consumer:** Преподаватель, аналитический модуль проекта.

## 2. Схема Mart (data/mart)
| Поле | Тип | Описание |
|---|---|---|
| `year` | int | Год |
| `value` | float | Значение выбросов |
| `name` | string | Название страны из countries.csv |
| `abs_change` | float | Изменение (текущий - прошлый год) |
| `growth_rate_pct` | float | % роста |

## 4. Допущения и единицы измерения
- **Единицы измерения:** Метрические тонны на душу населения.
- **Время:** Годовые показатели (агрегация за календарный год).
- **Валюта:** Не применимо.

## 5. Правила качества (DQ)
- **Uniqueness:** Сочетание `(country_iso3, indicator, year)` должно быть уникальным.
- **Completeness:** Допустимы пустые значения (`null`) в поле `value`, так как данные за некоторые годы могут отсутствовать в API.
- **Validity:** Значение `year` должно быть в диапазоне от 1960 до текущего года.
- **Freshness:** Наличие данных как минимум за последние 5 лет от текущей даты (с учетом задержки публикации).

## 6. Версионирование
- **Версия контракта:** 1.0.0.
- **Как фиксируете изменения:** Обновление файла в репозитории при изменении схемы API или логики обработки.
