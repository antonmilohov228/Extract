# Implementation Plan: Вариант 14 (World Bank CO2)

## 1. Краткое описание задачи

Проект направлен на автоматизацию сбора и анализа данных о выбросах $CO_2$ на душу населения в США. Данные извлекаются из публичного API Всемирного банка, проходят процесс очистки и трансформации для построения аналитических отчетов о динамике экологии за последние 30-60 лет.

## 2. Цель и ожидаемый результат

* **Файлы:** Сбор сырых данных в формате JSON, создание очищенного набора данных в CSV/Parquet.
* **KPI:** * Последнее доступное значение и год.
* Изменение показателя за 10 лет: 
Delta_{10y} = V_{year} - V_{year-10}


* Среднее значение (avg) за последние 10 лет.
* Экстремумы (Min/Max) за весь период.



## 3. Контуры системы (high level)

Источник (World Bank API) $\rightarrow$ Staging (`data/raw/*.json`) $\rightarrow$ Transform (Python/Pandas) $\rightarrow$ Normalized Layer (`data/processed/*.csv`) $\rightarrow$ Отчет/Дашборд.

## 4. Схема данных

### 4.1 Raw слой

* **Файл:** `worldbank_usa_co2_YYYYMMDD.json`
* **Ключи:** Массив объектов, где каждый объект содержит поля `indicator`, `country`, `value`, `date`.
* **Пример записи:** `{"indicator": {"id": "EN.ATM.CO2E.PC", ...}, "country": {"id": "US", "value": "United States"}, "value": 14.67, "date": "2020"}`

### 4.2 Normalized слой

* **Таблица:** `fact_co2_emissions`
* **Поля:** `year` (int), `value` (float), `country_iso3` (string), `indicator` (string).

### 4.3 Mart слой (агрегаты)

* **Таблица:** `agg_co2_stats`
* **KPI:** `latest_value`, `delta_10y`, `avg_10y`, `min_year`, `max_year`.

## 5. Data Quality (DQ)

* **NOT NULL:** `year`, `country_iso3`, `indicator`.
* **Диапазоны:** `year` >= 1960 и =< 2026.
* **Уникальность:** Составной ключ `(country_iso3, indicator, year)`.
* **Freshness:** Проверка наличия данных как минимум за позапрошлый год (учитывая лаг публикации WB).

## 6. Инкрементальность и идемпотентность

* **Повторный запуск:** Скрипт Extract перезаписывает файл за текущую дату или создает новый с временной меткой, обеспечивая актуальность.
* **Дубликаты:** На этапе Transform реализована проверка `drop_duplicates` по ключу `year`.
* **Watermark:** Используется фильтр по дате в параметрах API (при необходимости).

## 7. План тестирования

* Проверка корректности обработки `null` в поле `value`.
* Тест на обрыв соединения с API (обработка таймаутов).
* Проверка соответствия типов данных после парсинга JSON.

## 9. Риски и ограничения

* **Лаг данных:** Всемирный банк публикует данные с задержкой в 2-3 года.
* **Лимиты:** API World Bank может ограничивать частоту запросов (решается таймаутами).
* **Пропуски:** Исторические данные могут содержать `null` за ранние годы.
